Part 1 - Tracing and Commenting (80 pts)

I have given you some code. If you run the code here you will see it asks for an input, then nothing appears to happen after that. I will tell you that the input you give, is the velocity to throw a ball directly up into the air. What the code actually does is for you to figure out. Read through the code, and figure out what it's doing.

As you read through this code, comment it. What is each function doing? What is each line doing? How does this line help us reach the end goal? What does function mean? What does each line mean? I have intentionally used single letter variable names. What is each variable used for? These are some questions I want you to keep in mind.

When commenting your code, I want you to actually think and dig deep. Let's consider the following fragment of code. This code is a simple counting program that counts from 1 to 5. 

var c
c : 1
loop {
	until .c > 5
	iprint(.c)
	nl()
	c : .c + 1
}

Now one way to comment this would be the following. 

var c # initializes variable c
c : 1 # sets c to 1
loop {
	until .c > 5 # loops until the contents of c are greater than 5
	iprint(.c) # prints the contents of c
	nl() # prints a nl
	c : .c + 1 # adds 1 to the contents of c and updates the value
}

Let me tell you why I hate the above. While it does accurately explain what the code is doing, it has no relation to what the code means. Here's a better example. 

var c # initializes variable c. c represents the current count
c : 1 # sets c to 1, the count starts at 1

# loops until we reach the end of our count, which is when the contents of c are
# greater than 5
loop {
	until .c > 5
	iprint(.c) # prints the current count
	nl() # prints a nl for clarity
	c : .c + 1 # increases the count by adding 1
}

Now as you can see, the comments are a LOT more descriptive, and a lot more helpful. You don't need to comment every single line of code, but definitely consider the questions in the paragraph above, and comment what you think is necessary.

I will be grading you based on how helpful and accurate your comments are. That being said, there is no "correct answer". I don't have a rubric of comments I expect you to have. If you have the exact same character for character comments as someone else, you'll both get 0s for plagiarism. You won't need to work with a partner for this lab because this lab is about reading and comprehending code, which is done individually. Commenting code is aimed at helping YOU understand what's going on.
